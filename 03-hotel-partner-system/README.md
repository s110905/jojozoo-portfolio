## 📖 專案概述 (Project Overview)
本專案是為園區合作飯店開發的 **B2B 票券管理平台**。為了解決過去實體券核銷繁瑣、人工對帳數據不精確的痛點，系統實現了從「飯店端自主產券」到「園區端即時核銷」的全數位化流程。核心亮點在於支援「單張 QR 多人、分次核銷」的複雜邏輯，極大提升了入園效率與數據透明度。

## 🚩 B2B 合作的營運瓶頸
在與飯店合作發放票券時，過往的作業模式存在以下風險：
*   **核銷難度高**：園區現場難以辨識飯店發出的實體憑證真偽，且無法追蹤已使用人數。
*   **結算爭議多**：由於缺乏即時數據，雙方在月底對帳結算時常因「入園人數」認知不一而產生爭議。
*   **體驗不佳**：多人同行的家庭若分批入園，傳統票券難以處理「部分核銷」。

## 🛠️ 技術亮點 (Technical Highlights)

*   **全棧型別安全 (End-to-End Type Safety)**：採用 **React 18 + TypeScript + Supabase**，結合 Zod 進行嚴謹的資料校驗，確保系統的高穩定性。
*   **靈活的額度核銷引擎**：
    *   **多人分次使用**：單張 QR 可設定總人數（如家庭票），支援分批入園核銷。
    *   **原子更新 (Atomic Updates)**：確保在高併發環境下，剩餘名額扣減絕對精確。
*   **權限控管 (RBAC)**：嚴謹的角色區分，確保飯店夥伴、掃碼員工與管理員僅能存取授權範圍內的數據。
*   **效期管理**：自動化的有效期檢核機制，降低逾期票券誤用的營運風險。
