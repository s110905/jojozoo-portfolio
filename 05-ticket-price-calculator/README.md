## 🚩 銷售與服務痛點 (Pain Points)
在樂園售票第一線，傳統的售票模式面臨以下「服務摩擦」：
*   **決策遲疑**：面對複雜的優惠組合（壽星、地區、聯名卡），遊客往往在窗口猶豫不決，導致排隊隊伍過長。
*   **資訊不對稱**：遊客擔心自己沒買到最划算的票種，產生負面消費情緒。
*   **售票壓力**：第一線人員需背誦數十種折扣規則，並在壓力下進行手動試算，極易出錯。

## 🎯 產品策略：提升「消費爽感」與「結帳效率」 (Strategy)
本工具的核心目標是透過 **「透明化的決策支援」** 來優化銷售漏斗：
*   **即時最優解**：利用演算法一鍵找出最低價組合，給予遊客「賺到了」的正面心理暗示，加速購票決策。
*   **數位對話工具**：這不只是計算機，而是遊客與售票員之間的「溝通橋樑」，降低現場溝通成本。
*   **極簡介面 (Frictionless UI)**：針對現場情境設計，遊客在排隊時即可於手機完成試算，到窗口直接出示結果。

## 📈 商業價值與影響力 (Business Impact)
*   **提升成交速度**：預期縮短單筆購票溝通時間 **30-50%**，大幅增加假日尖峰時段的入園吞吐量。
*   **增加客單價**：透過清晰的組合試算，更有機會促成加購或升級套票。
*   **降低培訓成本**：新進人員不需死背複雜票價，即可提供專業的諮詢服務。

*   **複雜折扣演算**：自動處理「當日/當月壽星」及其「陪同者」的折扣連動。
*   **地區限定優惠**：動態整合「南投縣民」、「草屯鎮民」及「台中通 APP」等地區性促銷。
*   **節慶自動導航**：內建兒童節連假等特定日期的特殊票價規則，不需人工翻查表單。
*   **決策效率化**：為購票者提供清晰的費用拆解 (Breakdown)，並具備「出示售票員」功能，縮短窗口溝通時間。

## ✨ 核心功能 (Core Features)

- **智慧組合推薦**：系統會自動嘗試所有排列組合，確保產出總金額最低的購票方案。
- **動態日期感應**：依據選擇的遊園日期（平日、假日、特定節慶），自動啟用或關閉對應的優惠選項。
- **視覺化拆解**：清晰顯示每張票種選用的優惠類型與原始票價對比。
- **響應式 UI**：採用毛玻璃質感 (Glassmorphism) 設計，適合行動裝置於入園現場即時使用。

## 🛠️ 技術亮點 (Technical Highlights)

*   **React 18**: 使用現代 React 鉤子 (`useMemo`, `useState`) 實現高效能的實時試算引擎。
*   **演算法優化**: 實作貪婪演算法 (Greedy Algorithm) 概念，優先分配高面額折扣與壽星陪同額度，確保總額最優。
*   **設計美學**: 結合 Lucide React 圖標與動態 CSS 動畫，提供精品級的使用體驗。
